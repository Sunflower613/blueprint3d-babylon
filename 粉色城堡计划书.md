# 粉色城堡与家园沙盘计划书

本计划书旨在明确 `blueprint3d-babylon` 建筑编辑库在“粉色城堡”与“家园沙盘”项目中的具体应用场景、对接方式及后续的业务实施路线图。

---

## 业务背景与愿景

“粉色城堡”及“家园沙盘”项目致力于为用户提供一套高自由度、高视觉表现力的梦幻建筑设计与互动系统。
通过引入 `blueprint3d-babylon` 底层编辑库，用户可以像玩《模拟人生 4》一样：
1. 自定义绘制梦幻城堡的房间平面图，动态拖动并调整城堡房间长宽；
2. 随意开凿不同款式的门窗，摆放梦幻家具（如公主床、壁炉、秋千等）；
3. 设定具有粉色调、欧式复古调的墙面、地板与家具组件材质；
4. 将生成的城堡建筑文件加载入玩家运行场景，成为可供角色行走、探索、碰撞互动的实体游戏大地图。

---

## 业务对接与用法

库在 `src/presets/pinkCastle.js` 及 `src/runtime/PinkCastleGenerator.js` 中保留了对粉色城堡核心场景的兼容接口。

### 1. 兼容原有 CastleGenerator 用法
在游戏运行时场景中，可通过 `PinkCastleGenerator` 快速初始化城堡：

```javascript
import { PinkCastleGenerator as CastleGenerator } from './blueprint3d-babylon/src/index.js';
import { Player } from './game/Player.js'; // 游戏业务逻辑中的玩家类

// 初始化城堡生成器
const castle = new CastleGenerator(scene, themeConfig);

// 将城堡的碰撞体网格 (castle.colliders) 传入玩家实例中，实现障碍物阻挡与行走限制
const player = new Player(scene, camera, castle.colliders, themeConfig);

// 在游戏主循环中更新城堡与玩家状态
function gameLoop(delta, time) {
  castle.update(delta, time, player);
  player.update(delta);
}
```

### 2. 底层蓝图式直接构建
若需在 Babylon 场景中通过参数化脚本直接构建一个定制配色的粉色城堡，可调用底层快捷构建函数：

```javascript
import { buildPinkCastle } from './blueprint3d-babylon/src/index.js';

// 传入指定的调色盘与场景上下文
const castle = buildPinkCastle(scene, {
  palette: {
    wall: '#ff8fb3', // 主粉色墙体
    roof: '#b71c1c', // 尖顶城堡红色屋顶
    trim: '#ffffff'  # 白色包边
  }
});
```

## 实施路线图与后续计划

### 第一阶段：业务包解耦与配置化
1. **城堡专用家具资源归档**：将属于粉色城堡特有的建筑装饰（如城堡尖顶、护栏、回音廊柱等）以及预设的粉色蕾丝织物、粉色大理石材质统一归纳到 `src/presets/pinkCastle.js`。
2. **材质资源 ID 化**：当前自定义图片材质存在建筑文件中时使用了 Base64 格式，导致 JSON 导出文件臃肿。后续将建立默认材质包的资源服务器或本地资源 ID 映射表，在保存文件时只引用材质 ID（例如 `"materialId": "pink_marble_01"`）。

### 第二阶段：沙盘编辑器集成
1. **2D/3D 沙盘编辑**：搭建游戏内的家园编辑面板，用户在沙盘中通过 2D 视角画好户型、摆放完家具后，一键生成 `*.b3dbuilding.json`，上传至后端服务器。
2. **动态加载与碰撞烘焙**：在玩家访问好友家园或进入自己的城堡时，运行时系统自动请求对应的 `.json` 文件，调用 `Blueprint3DTestMap.loadBuildingFile()` 进行三维重建，并同步更新物理引擎碰撞体。

### 第三阶段：独立仓库托管与组件化
1. **独立仓库托管**：当底层的墙体裁剪、几何生成、DXF/3MF 导出和性能优化等技术点稳定后，将 `blueprint3d-babylon/` 作为独立的前端基础库上传至 GitHub。
2. **NPM 依赖化管理**：通过 npm 将其作为独立依赖引入到主游戏项目中，避免业务层代码与底层 3D 渲染几何计算的逻辑混杂，提高整体项目的可维护性与协作开发效率。
